Blinky

  • Date:8/1/2012
  • Plateform:iOS
  • Framework:UIKit
  • Language:Objective-c

Blinky is a simple flashlight app. Yes, another flashlight app to add to the hundreds of other flashlight apps on the App Store. I tried to make Blinky unique by adding a feature that will covert the user's text input into Morse code through the iPhone's LED. I found out later that there are hundreds of these apps as well.

I mainly made this app since it was simple and my only experience in iOS development was a game for a class project. The game used Cocos2d, an open source framework, so I had no experience with UIKit or CocoaTouch. My main goal in developing this app was to gain experience with:

  • ViewControllers
  • NSUserDefaults
  • Updating IBOutlets
  • Connecting actions to UIButton
  • NSTimer
  • Populating an NSDictionary from a plist
  • Keyboard inputs
  • UI Customization (realizing I'm terrible at design) ---->
  • App submission process

Still waiting for Blinky to be reviewed.


Update (8/20/12): Blinky was rejected.

DomainFinder

  • Date:3/24/2012
  • Plateform:Web Application
  • Framework:JavaServlets
  • Language:Java and Javascript

DomainFinder is a Java servlet web app that generates a combination of domain names and checks to see if the domains are available.

The application works fine, but I lost interest in the project when I found out that running the web app on a Java server would be expensive. There are not many cheap java servers and most of them do not allow spawning runtime threads like Google App Engine. My Java Servlet creates a runtime thread to run the whois command. I might look into a tiny Rackspace server to test out how much it would cost me. Also I plan on learning Python + Django over the summer so I might covert this project.

Clumsy Chef

  • Date:11/20/2011
  • Plateform:iOS
  • Framework:Cocos2D
  • Language:Objective-c

ClumsyChef is an iOS game that I developed, for a game development class, with three other students. It is a vertical side-scrolling game using the iPhone's accelerometer to move the player side to side. The goal of the game is go collect recipe items within a short time limit.

My role was to write game logics for the game layer. This included collision detection, checking win/lose conditions, and generating different game level items. The game is not on the App Store because we used content that we did not have any rights to. No one on our team knew how to do any art/music.
